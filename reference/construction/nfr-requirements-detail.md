# NFR Requirements — Detailed Steps

**Purpose**: Determine non-functional requirements and tech stack for the unit.

## Prerequisites
- Functional Design complete for this unit

## Step 1: Analyze Functional Design
Read artifacts from `aidlc-docs/construction/{unit-name}/functional-design/`.

## Step 2: Create Plan
Generate plan with checkboxes for NFR assessment.

## Step 3: Generate Questions
Proactive questioning across ALL categories:
- Scalability: load, growth, capacity planning
- Performance: response times, throughput, latency
- Availability: uptime, disaster recovery, failover
- Security: data protection, compliance, auth
- Tech Stack: preferences, constraints, integration
- Reliability: fault tolerance, monitoring, alerting
- Maintainability: code quality, testing, operations
- Usability: UX, accessibility

## Step 4: Store Plan
Save as `aidlc-docs/construction/plans/{unit-name}-nfr-requirements-plan.md`.

## Step 5: Collect and Analyze Answers
Wait → Review → Follow up → Resolve.

## Step 6: Generate Artifacts
In `aidlc-docs/construction/{unit-name}/nfr-requirements/`:
- nfr-requirements.md
- tech-stack-decisions.md

## Step 7: Present Completion
```markdown
# NFR Requirements Complete - [unit-name]

[AI summary]

> **REVIEW REQUIRED:**
> `aidlc-docs/construction/[unit-name]/nfr-requirements/`

> **WHAT'S NEXT?**
> - Request Changes
> - Continue to Next Stage — proceed to **[next stage]**
```

Wait for approval. Log in audit.md. Update state.
